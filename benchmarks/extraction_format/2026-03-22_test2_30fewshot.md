# Test 2 — 30 few-shot couvrant toutes les hypostases

**Date** : 2026-03-22
**Texte source** : "La Ronde des Intelligences" (page 15, 19726 chars, ~13 chunks)
**Différence avec le test 1** : few-shot passent de 2 extractions à 30 (une par hypostase),
prompt amélioré avec définitions par familles épistémiques.

---

## 1. Changements par rapport au test 1

| Élément | Test 1 (2 few-shot) | Test 2 (30 few-shot) |
|---|---|---|
| **Nombre de few-shot** | 2 extractions | 30 extractions (une par hypostase) |
| **Texte few-shot** | Texte court sur l'IA (2 phrases) | Texte synthétique sur l'IA dans l'éducation (6 paragraphes) |
| **Définitions** | Liste plate des 30 noms | 6 familles épistémiques avec définition informelle + formelle |
| **Classes dans few-shot** | Approche B : 2 classes (théorie, problème) | Approche B : 30 classes (toutes les hypostases) |
| **Prompt instructions** | Basique | Amélioré (instructions proportionnelles à la longueur du texte) |

## 3. Analyse comparative Test 1 vs Test 2

### 3.1 Impact des 30 few-shot sur Gemini

| Métrique | Test 1 | Test 2 | Analyse |
|---|---|---|---|
| Classes en approche B | 2 (théorie 87%, problème 13%) | **20 classes équilibrées** | Gemini copie les few-shot — avec 30 exemples il diversifie |
| Distribution | Très déséquilibrée | `définition` 13%, `principe` 12%, `méthode` 10%... | Beaucoup plus homogène |
| Volume | 112 extractions | 90 extractions | -20% (le prompt plus long consomme du contexte) |
| Couverture attributs | 28/30 | 27/30 | Stable |

### 3.2 Impact des 30 few-shot sur GPT

| Métrique | Test 1 | Test 2 | Analyse |
|---|---|---|---|
| Classes en approche B | 14 | **23** | GPT diversifie aussi, mais hallucine plus |
| Hallucinations classe | 0 | 5 | Le grand nombre de classes augmente le risque d'invention |
| Volume | 95 extractions | 87 extractions | -8% (impact modéré du prompt plus long) |
| Couverture attributs | 26/30 | 25/30 | Stable |

### 3.3 Stabilité

**Les 4 tests produisent 0 crash.** Pas de boucle de répétition, pas de JSON corrompu.
Le prompt amélioré (définitions par familles + 30 few-shot) est aussi stable que le prompt minimal.

### 3.4 Coût du prompt plus long

Le passage de 2 à 30 few-shot augmente la taille du prompt d'environ 8K tokens. Impact :
- **Volume** : ~15% d'extractions en moins (90-102 vs 95-118)
- **Temps** : comparable ou légèrement plus rapide (le LLM "comprend" mieux ce qu'on attend)
- **Qualité** : classification beaucoup plus riche

## 4. Synthèse globale (Test 1 + Test 2)

### 4.1 Résultats clés

1. **Le biais des few-shot est confirmé** : Gemini reproduit les classes vues dans les exemples. Avec 2 few-shot il produit 2 classes, avec 30 il en produit 20.

2. **La couverture dans les attributs est stable** (~27/30) quel que soit le nombre de few-shot. Les attributs portent la richesse sémantique indépendamment du format.

3. **GPT extrapole, Gemini copie** : GPT produit 14 classes même avec 2 few-shot, Gemini a besoin de voir chaque classe.

4. **Le prompt par familles épistémiques stabilise les deux modèles** : 0 erreur sur les 8 tests (4 du test 1 + 4 du test 2).

5. **Gemini hallucine très peu**, GPT hallucine plus mais les hallucinations sont filtrables par la normalisation.

### 4.2 Recommandation

**Approche B avec 30 few-shot** est la meilleure combinaison :
- Gemini produit 20 classes distinctes (vs 2 avec 2 few-shot)
- GPT produit 23 classes (vs 14)
- La couverture dans les attributs reste bonne (25-27/30)
- Zéro crash, zéro boucle
- Le coût du prompt plus long (-15% extractions) est compensé par la richesse de classification

**Pour la production**, les 30 few-shot devraient être stockés dans la DB
comme `AnalyseurExample` + `ExampleExtraction`, et le prompt utiliser les
définitions par familles épistémiques.
